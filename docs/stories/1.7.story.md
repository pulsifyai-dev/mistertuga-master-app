## Story

**As a** developer,
**I want** the Cloud Function replaced with a Supabase database webhook, the UI language standardized, and the auth provider fully integrated,
**So that** all remaining integration points are connected before end-to-end testing.

---

## Context

This story bundles three smaller integration tasks that depend on earlier stories:
1. **Webhook migration** (TD-C6 related): Replace Firebase Cloud Function with Supabase `notify_tracking_added()` trigger (Appendix C SQL).
2. **Language standardization** (TD-M11): Standardize all UI text to English (pending decision D6).
3. **Auth provider update**: Replace Firebase AuthProvider with Supabase Auth context.
4. **Additional security hardening**: Rate limiting (TD-M2), webhook URL validation (TD-M3), dynamic import for heavy libraries (TD-M10), inline style cleanup (TD-M14).

**Reference:** `docs/architecture/technical-debt-assessment.md` Appendix C, `docs/architecture/ux-specialist-review.md` §5

---

## Acceptance Criteria

- [ ] **AC1:** Database webhook trigger replaces Cloud Function
  - `notify_tracking_added()` function deployed (Appendix C SQL)
  - Trigger fires on `orders.tracking_number` update (NULL → value)
  - Webhook sends JSON payload to configured URL
  - pg_net extension verified enabled
  - Test: Update tracking number → webhook fires within 5s

- [ ] **AC2:** UI language standardized to English (if D6 = English)
  - All Portuguese strings replaced with English equivalents
  - Button labels, form labels, table headers, error messages
  - Files affected: sidebar, header, orders page components, settings
  - No hardcoded Portuguese remaining in UI

- [ ] **AC3:** AuthProvider updated for Supabase
  - Firebase AuthProvider replaced with Supabase Auth context
  - Session management via Supabase client
  - User role available in auth context
  - Login/logout flows work end-to-end

- [ ] **AC4:** Rate limiting on Server Actions (TD-M2)
  - Basic rate limiting on write operations
  - Prevent rapid-fire mutations

- [ ] **AC5:** Webhook URL validation (TD-M3)
  - Validate URL format before saving to settings
  - Prevent SSRF: block internal/private IP ranges
  - Only allow HTTPS URLs

- [ ] **AC6:** Heavy libraries dynamically imported (TD-M10)
  - PDF library (`jspdf`, `jspdf-autotable`) loaded via `dynamic import()`
  - Excel library (`exceljs`) loaded via `dynamic import()`
  - Reduces initial bundle by ~800KB

- [ ] **AC7:** Inline styles replaced with Tailwind classes (TD-M14)
  - Remove `style={{}}` overrides
  - Use Tailwind utilities or CSS variables from design tokens
