## Context

This story bundles three smaller integration tasks that depend on earlier stories:
1. **Webhook migration** (TD-C6 related): Replace Firebase Cloud Function with Supabase `notify_tracking_added()` trigger (Appendix C SQL).
2. **Language standardization** (TD-M11): Standardize all UI text to English (pending decision D6).
3. **Auth provider update**: Replace Firebase AuthProvider with Supabase Auth context.
4. **Additional security hardening**: Rate limiting (TD-M2), webhook URL validation (TD-M3), dynamic import for heavy libraries (TD-M10), inline style cleanup (TD-M14).

**Reference:** `docs/architecture/technical-debt-assessment.md` Appendix C, `docs/architecture/ux-specialist-review.md` §5

## Dev Notes

- **AC3 was already done**: Auth was migrated in Story 1.2 (auth-provider.tsx, middleware.ts, auth.ts all use Supabase). Firebase files still exist but only profit-stats/page.tsx uses them (deferred to EPIC-2 Story 2.7).
- **Appendix C SQL had 2 bugs**: Referenced `app_settings` table (actual: `settings`) and key `tracking` (actual: `webhook_url`). Fixed in migration.
- **No exceljs in codebase**: AC6 spec mentioned exceljs but it's not imported anywhere. Only jsPDF was converted to dynamic import.
- **Inline styles**: Only 2 static inline styles could be fully converted to Tailwind. Dynamic runtime styles (country colors, expense accent colors, CSS variables in shadcn/Recharts) must remain as inline style or CSS custom properties — this is standard practice.
- **Rate limiter is per-process**: In Vercel serverless, each function instance has its own Map. For stricter rate limiting, upgrade to Upstash Redis in a future story.
- **TypeScript**: Clean compilation (0 errors)
