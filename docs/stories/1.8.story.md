**Epic:** EPIC-1 (Firebase → Supabase Quality Migration)
**Sprint:** 3 (Days 18-19)
**Effort:** 1.5 days
**Priority:** P0
**Status:** Draft

## Story

**As a** QA engineer,
**I want** comprehensive end-to-end testing of all application flows on the new Supabase backend,
**So that** we have confidence the migration is complete and no functionality was lost.

---

## Context

With all data migrated (Story 1.3), hooks rewritten (Story 1.5), accessibility fixed (Story 1.6), and integrations complete (Story 1.7), this story validates the entire application works correctly on Supabase before deploying.

**Reference:** `docs/architecture/qa-review.md` §5.3, `docs/architecture/technical-debt-assessment.md` §3.3 Phase D

---

## Acceptance Criteria

- [ ] **AC1:** All ADMIN user flows verified
  - Login as ADMIN
  - View orders across all countries
  - Create, edit, delete order
  - Update tracking number → webhook fires
  - Export PDF and Excel
  - View profit stats
  - Manage expenses
  - Update app settings
  - Manage users

- [ ] **AC2:** All FORNECEDOR (supplier) user flows verified
  - Login as FORNECEDOR
  - View orders for assigned countries only
  - Cannot edit or delete orders
  - Cannot access settings, profit stats, expenses
  - Cannot access user management

- [ ] **AC3:** Real-time updates verified
  - Two browser windows open
  - Admin updates order in window 1 → window 2 reflects change within 2s
  - New order appears in both windows

- [ ] **AC4:** Accessibility verified
  - Lighthouse accessibility score > 90
  - axe-core: 0 critical or serious violations
  - Keyboard navigation: all flows completable via keyboard
  - Skip navigation link works
  - Screen reader reads all content meaningfully

- [ ] **AC5:** Data integrity verified post-migration
  - Spot-check 20 orders: all fields correct
  - Customer data matches original Firestore records
  - Profit stats totals match original values
  - No orphaned records in any table

- [ ] **AC6:** Error handling verified
  - Network error → error boundary shows retry
  - Auth expired → redirect to login
  - RLS violation → appropriate error message
  - Empty data → empty states shown

- [ ] **AC7:** Performance acceptable
  - Initial page load < 3s
  - Order list load < 1s
  - Real-time update latency < 2s
  - PDF export < 5s for 100 orders
  - No memory leaks on long sessions

- [ ] **AC8:** Test foundation established (TD-H3 partial)
  - Key test scenarios documented
  - Critical path smoke test script created
  - Regression test checklist for future changes
