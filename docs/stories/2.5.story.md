## Story

**As an** admin user,
**I want** Google Ads and Meta Ads spend data automatically fetched and displayed in the profit dashboard,
**So that** I have a complete view of revenue vs all expenses without checking multiple platforms.

---

## Context

The profit dashboard currently shows Shopify revenue + manual expenses. Ad spend is the largest operational cost and must be integrated for accurate profit calculation. Both platforms offer APIs for fetching daily spend data. The n8n "Faturamento" workflow already references a Facebook Ad Account ID and Shopify store config.

**Reference:** `docs/product/mistertuga_master.md` §2.1-2.3
**Known config:** Meta Ads account ID: referenced in n8n workflow

## Dev Notes

- **No npm dependencies added.** All API calls use native `fetch()`.
- **Google Ads** uses non-streaming `search` endpoint (not `searchStream`) for simpler response parsing with pagination support.
- **OAuth2 token refresh** implemented inline — exchanges refresh token for access token on each sync (tokens are short-lived, sync runs at most daily).
- **Meta Ads** normalizes account ID (strips `act_` prefix if present) and follows `paging.next` for pagination.
- **Shared sync logic** in `ad-sync.ts` is used by both the server action (`syncAdAccountSpend`) and the cron route, avoiding code duplication.
- **Tasks 11-12** require real API credentials to test end-to-end. The code gracefully returns descriptive error messages when env vars are missing.
- **Vercel Cron** requires Pro plan. On Hobby plan, use the manual "Sync Now" button as fallback.
