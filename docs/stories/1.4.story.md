# Story 1.4: Orders Page Decomposition

**Epic:** EPIC-1 (Firebase → Supabase Quality Migration)
**Sprint:** 2 (Days 8-11)
**Effort:** 3-4 days
**Priority:** P1
**Status:** Draft
**Agent:** @dev (Dex)
**Blocked By:** Story 1.0 (can start during Sprint 1)
**Blocks:** Stories 1.5, 1.6

---

## Story

**As a** developer,
**I want** the monolithic 1,704-line orders page decomposed into 9 modular components and 4 hooks,
**So that** the Supabase hook rewrite can be done safely on smaller, testable units.

---

## Context

`src/app/(app)/master-shopify-orders/page.tsx` is 1,704 lines with 23 `useState` hooks, mixing data fetching, filtering, UI rendering, export logic, and edit modals. Rewriting all Firebase hooks to Supabase in this monolith is high-risk. Decomposing first reduces the page to ~250 lines with ~4 state hooks, making the Supabase migration in Story 1.5 significantly safer.

**Reference:** `docs/architecture/technical-debt-assessment.md` Appendix B, `docs/architecture/ux-specialist-review.md` §4

**Important:** This story refactors the EXISTING Firebase code. The Supabase hook rewrite happens in Story 1.5. Keep all Firebase functionality working throughout.

## Acceptance Criteria

- [ ] **AC1:** Types extracted to shared `types.ts` file
  - Order, OrderItem, Customer, FilterState types defined
  - No inline type definitions remaining in page.tsx

- [ ] **AC2:** 3 atom components extracted
  - `OrderStatusBadge.tsx` (~30 lines) — status chip with color mapping
  - `TrackingInput.tsx` (~60 lines) — inline tracking number edit
  - `OrderEmptyState.tsx` (~40 lines) — zero results message

- [ ] **AC3:** 4 hooks extracted
  - `useOrders.ts` (~100 lines) — data fetching + real-time sync
  - `useOrderFilters.ts` (~80 lines) — filter state + logic
  - `usePdfExport.ts` (~120 lines) — PDF generation
  - `useExcelExport.ts` (~80 lines) — Excel generation

- [ ] **AC4:** 3 molecule components extracted
  - `OrderRow.tsx` (~150 lines) — single order row
  - `OrderFilters.tsx` (~150 lines) — search + date + status filters
  - `CountryTabs.tsx` (~80 lines) — country tab navigation

- [ ] **AC5:** 3 organism components extracted
  - `OrdersTable.tsx` (~300 lines) — table + pagination
  - `OrderEditDialog.tsx` (~200 lines) — edit form modal
  - `ExportControls.tsx` (~100 lines) — PDF + Excel buttons

- [ ] **AC6:** page.tsx reduced to orchestrator role
  - ~250 lines maximum
  - ~4 useState hooks (down from 23)
  - Composes extracted components
  - No business logic in page.tsx

- [ ] **AC7:** All existing functionality preserved
  - Real-time order updates still work
  - Country tab switching works
  - Search, filter, pagination work
  - PDF and Excel exports work
  - Order editing works
  - Role-based visibility works

## Risk Mitigation

| Risk | Mitigation |
|------|------------|
| Decomposition breaks real-time | Verify real-time updates after EACH extraction step (not just at the end) |
| State prop drilling | Use composition pattern — pass handlers, not state |
| Import cycle | Follow extraction order: types → atoms → hooks → molecules → organisms → page |
