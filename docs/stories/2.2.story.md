# Story 2.2: Exchanges/Returns — UI + Email Templates

**Epic:** EPIC-2 (Post-Migration New Features)
**Sprint:** 4
**Effort:** 2 days
**Priority:** P1
**Status:** Done
**Agent:** @dev (Dex) + @ux-design-expert (Uma)
**Blocked By:** Story 2.1

---

## Story

**As an** admin user,
**I want** a dedicated Exchanges tab with a data table and the ability to send templated emails to customers,
**So that** I can manage all exchange/return cases efficiently from the app.

---

## Context

With the exchanges schema and n8n webhook in place (Story 2.1), this story builds the UI. The exchanges table follows the same patterns established in EPIC-1 (decomposed components, skeleton loaders, empty states, accessibility). Email sending uses configurable templates with placeholder variables.

**Reference:** `docs/product/mistertuga_master.md` §4.3, `docs/architecture/schema-blueprint.md` (email_templates)

---

## Acceptance Criteria

- [x] **AC1:** Exchanges page accessible from sidebar
  - New sidebar item: "Exchanges" with RefreshCw icon
  - Route: `/exchanges`
  - aria-current on active link (follows EPIC-1 a11y patterns)

- [x] **AC2:** Exchanges data table
  - Card-based rows: Order #, Customer, Status, Reason, Created, Actions
  - Sortable columns (created_at, order_number, customer_name, status)
  - Filterable by status (7 tabs: All + 6 statuses)
  - Search across order_number, customer_name, customer_email
  - Skeleton loader while loading
  - Empty state: "No exchange requests" with description

- [x] **AC3:** Exchange detail view
  - Right-side Sheet (drawer) showing full exchange details:
    - Customer contact info (name, email, phone, address)
    - Original order details (if linked)
    - Received/reported description
    - Original email text (monospace, scrollable)
    - Attachments (file list with sizes)
    - Internal notes (editable textarea + save)
    - Email log (chronological with status badges)

- [x] **AC4:** Status management
  - Dropdown on each row + Select in detail sheet
  - Status change via server action (requireAdmin + rateLimit)
  - Real-time update (Supabase Realtime subscription)

- [x] **AC5:** Email templates management (Admin settings)
  - CRUD for email templates in Settings page
  - Template fields: name, subject, body
  - Supports placeholders: {{customer_name}}, {{customer_email}}, {{order_number}}, {{reason}}, {{status}}
  - Preview mode: render template with sample data

- [x] **AC6:** Send email action
  - Button on exchange row: "Send Email"
  - Opens Dialog: select template, preview rendered email, confirm send
  - Email sent via Resend API
  - Email logged in exchange_email_log table
  - Toast confirmation: "Email sent to {customer_email}"

- [x] **AC7:** Email templates + email log tables created
  - Migration 005: email_templates table
  - Migration 006: exchange_email_log table
  - RLS: Admin full CRUD, Fornecedor no access

## Dev Notes

- **UI Pattern:** Card-based rows (not HTML tables) following `OrderRow.tsx` pattern
- **Detail View:** shadcn Sheet (right drawer) with `sm:max-w-lg` override
- **Realtime:** Reuses exchanges channel from Story 2.1 (already enabled)
- **Email Service:** Resend API — server-side only (in server actions)
- **Template Engine:** Simple `string.replaceAll` for `{{placeholders}}` — no external library
- **RLS:** Admin-only on both new tables, matching exchanges pattern
- **Migrations:** Sequential timestamps `20260303150005-6`, idempotent
- **Build verified:** `npx tsc --noEmit` and `npm run build` pass clean
- **Migrations applied:** Both pushed to remote Supabase successfully
