## Story

**As a** developer,
**I want** all Firebase dependencies removed and the application deployed to the production hosting platform,
**So that** the codebase is clean and the migration is ready for go-live.

---

## Context

After QA gate passes (Story 1.8), this story removes all Firebase code, packages, and configuration. The application is deployed to the chosen hosting platform (pending decision D4, recommended: Vercel). Firebase is NOT decommissioned yet — it remains as a rollback target for 2 weeks.

**Reference:** `docs/architecture/technical-debt-assessment.md` §3.3 Phase D, §7 Rollback Strategy

---

## Acceptance Criteria

- [ ] **AC1:** All Firebase packages removed
  - `firebase` package removed from package.json
  - `firebase-admin` package removed from package.json
  - No Firebase imports in any source file
  - `npm install` succeeds without Firebase deps

- [ ] **AC2:** Firebase source files removed
  - `src/firebase/` directory removed (if exists)
  - `src/lib/firebase/` directory removed (if exists)
  - `functions/` directory removed (Cloud Functions)
  - Firebase config files removed (`.firebaserc`, `firebase.json`, `firestore.rules`, `firestore.indexes.json`)

- [ ] **AC3:** Environment variables updated
  - Firebase env vars removed from `.env` and hosting platform
  - Supabase env vars configured in hosting platform
  - No Firebase references in environment configuration

- [ ] **AC4:** Package cleanup (LOW items)
  - Package name fixed: "nextn" → "mistertuga" (TD-L1)
  - Empty AI flows file removed (TD-L2)
  - Placeholder images utility removed (TD-L3)
  - Custom shadows tokenized (TD-L6)
  - Unused Supabase link resolved (TD-L8)
  - Memoization added to heavy components (TD-L7)

- [ ] **AC5:** Application deployed to hosting platform
  - Build succeeds: `npm run build` passes
  - Lint passes: `npm run lint` passes
  - Type check passes: `npm run typecheck` passes
  - Deployed to production URL
  - SSL/HTTPS configured

- [ ] **AC6:** Firebase NOT decommissioned
  - Firebase project remains active
  - Firebase Hosting still serves (as rollback)
  - Firestore data intact
  - DNS/routing switched to new platform only
