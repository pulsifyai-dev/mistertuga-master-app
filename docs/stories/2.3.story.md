## Story

**As an** admin user,
**I want** production and shipping costs automatically calculated for each order based on configurable rate tables,
**So that** I can track supplier costs without manual calculations in external tools.

---

## Context

Currently, cost calculations happen in an Electron-based flow with Google Drive exports. This story rebuilds the calculation engine inside the app with database-backed cost rules. The cost structure involves:
- **Production cost**: per-item cost based on product type (currently ~$3-5/item)
- **Shipping cost**: per-item cost based on country + weight tiers
- Both costs are in USD (Chinese supplier), while revenue is in EUR

The "Master Costs" Excel shows reconciliation between system-calculated costs and Chinese supplier invoices, with a "Gap" column tracking discrepancies.

**Reference:** `docs/product/mistertuga_master.md` §5, `docs/architecture/schema-blueprint.md` (cost_rules, order_costs)
**Data source:** `docs/product/🔴 Misterfutbol_Master_Costs.xlsx` (reference for cost structure)

## Acceptance Criteria

- [ ] **AC1:** Cost rules table created via migration
  - Schema matches blueprint: country_code, rule_type (production/shipping), base_rate, rate_tiers (JSONB)
  - Admin UI to manage cost rules
  - Support for per-country rules

- [ ] **AC2:** Order costs table created
  - Stores computed: production_cost, shipping_cost, total_cost per order
  - Links to cost_rule used for calculation
  - Currency field (default USD)

- [ ] **AC3:** Cost calculation engine
  - Server Action: `calculateOrderCosts(orderId)`
  - Looks up applicable cost rules for order's country
  - Calculates per-item production cost (base_rate * quantity)
  - Calculates shipping cost from rate tiers
  - Stores result in order_costs table
  - Returns computed values

- [ ] **AC4:** Automatic cost calculation on new orders
  - When new order is ingested from Shopify, trigger cost calculation
  - Batch calculation support for bulk imports
  - Skip if cost rules not configured for country (log warning)

- [ ] **AC5:** Cost rules management UI (Admin)
  - Page in Settings: "Cost Rules"
  - CRUD for production rules per country
  - CRUD for shipping rules per country with tier editor
  - Rate tier editor: min_qty, max_qty, rate per tier
  - Preview: estimate cost for sample order

- [ ] **AC6:** Cost summary visible on order detail
  - Order detail view shows: production cost, shipping cost, total cost
  - Currency displayed (USD)
  - "Not calculated" state if no rules available

- [ ] **AC7:** RLS policies
  - Cost rules: Admin full CRUD, Fornecedor no access
  - Order costs: Admin read, Fornecedor no access
