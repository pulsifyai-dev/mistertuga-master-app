## Story

**As an** admin user,
**I want** to generate supplier export Excel files with embedded product images directly from the app,
**So that** I no longer need the Electron app or Google Drive for supplier order documents.

---

## Context

The current supplier export flow uses n8n + Google Drive to create Excel files for the supplier. The target flow generates the file entirely in-app using ExcelJS, with checkpoint-based export tracking to prevent duplicates. Critical requirement: product images must be embedded in the Excel (not just URLs).

**Reference:** `docs/product/mistertuga_master.md` §5.3-5.5, n8n workflow "Fluxo para Enviar Artigos para o Supplier"
**Export columns:** Order Name, Name of the Item, Quantity, Customer Name, Address, Size, Customization, Version, ThumbnailUrl (embedded)

## Dev Notes

- **ExcelJS dynamic import**: Used `await import('exceljs')` inside the server action to avoid bundling the library in the client. ExcelJS chosen over xlsx because it supports image embedding.
- **Image embedding via base64**: Images are fetched server-side, converted to base64, and embedded in cells. Fallback: URL text if fetch fails. 5-second timeout per image.
- **Checkpoint flow**: Uses `export_checkpoints` table with `UNIQUE(country_code, export_type)`. After export, upserts checkpoint with last_order_number. Next export only fetches orders after that number.
- **Cost per-item split**: Order-level costs from `order_costs` are split evenly across items for per-row display. Summary row shows actual totals.
- **Storage**: Files stored in `supplier-exports` bucket at `{country}/{fileName}`. Download via 5-minute signed URLs.
- **Export naming**: `{CC}_orders_{date}_{firstOrder}-{lastOrder}_{totalPcs}pcs.xlsx`
