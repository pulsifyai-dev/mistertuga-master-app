### Circular Dependency Resolution
`countries.shopify_store_id → shopify_stores(id)` and `shopify_stores.country_code → countries(code)` are circular. Resolved via 3-step migration:
1. Create countries WITHOUT shopify_store_id
2. Create shopify_stores WITH country_code FK
3. ALTER TABLE countries ADD shopify_store_id FK + backfill

### Audit Trigger Design
- Generic `audit_trigger_func()` uses `TG_TABLE_NAME` and `auth.uid()`
- `SECURITY DEFINER` allows the trigger to write to audit_log even when RLS is active
- Captures full row as JSONB (old_data/new_data)

### Updated_at Auto-Trigger
- Separate from audit — lightweight trigger that sets `updated_at = now()` on UPDATE
- Applied to: users, shopify_stores, customers, orders, expenses

### Note on Docker
- Docker Desktop not running — `supabase db dump` requires Docker
- All operations done via `supabase db push` (connects to remote directly)
- Schema verified via `supabase inspect db table-stats`
