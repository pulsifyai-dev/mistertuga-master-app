## Story

**As a** developer,
**I want** the complete Supabase database schema created with migrations, audit logging, and proper constraints,
**So that** auth migration and data loading can proceed on a solid foundation.

---

## Context

The target Supabase schema was designed in Phase 2 (`SCHEMA.md` §7) and refined in Phase 5 (`db-specialist-review.md`). This story implements the full PostgreSQL schema with all Phase 5 corrections: UNIQUE constraint on customer phone, timestamps on products, CHECK constraint on order IDs, and the audit_log table with triggers.

**Reference:** `docs/architecture/SCHEMA.md` §7, `docs/architecture/technical-debt-assessment.md` Appendix A
**Schema Blueprint:** `docs/architecture/schema-blueprint.md` — complete schema design for EPIC-1 + EPIC-2 tables

---

## Acceptance Criteria

- [ ] **AC1:** Supabase project created and configured
  - Project region: europe-west (matching current Firestore location)
  - Free tier limits verified: 500MB storage, 200 concurrent connections
  - Environment variables documented

- [ ] **AC2:** All 12 tables created via migration scripts
  - `countries`, `shopify_stores`, `users`, `customers`, `orders`, `order_items`, `settings`, `expenses`, `audit_log`, `export_checkpoints`
  - Plus `products` and `profit_stats` if needed per current Firestore schema
  - All foreign keys, CHECK constraints, NOT NULL constraints in place
  - Phase 5 corrections applied:
    - `UNIQUE(phone)` on customers
    - `created_at` + `updated_at` on products table
    - `CHECK` constraint on orders.id format
    - UUID PK for expense_categories (if decided)

- [ ] **AC3:** Audit log table + triggers operational
  - `audit_log` table created per Appendix A spec
  - Trigger functions on `orders`, `users`, `app_settings`
  - Test: Insert an order → audit_log row appears with correct data

- [ ] **AC4:** Indexes created for access patterns
  - Orders: `country_code`, `status`, `created_at`
  - Customers: `phone` (unique), `country_code`
  - Order items: `order_id`
  - Audit log: `table_name + record_id`, `user_id`, `created_at DESC`

- [ ] **AC5:** All migrations are versioned and reversible
  - Migration files in `supabase/migrations/`
  - Each migration has a corresponding rollback strategy
  - `supabase db push` applies cleanly

- [ ] **AC6:** Supabase Realtime enabled on orders table
  - Publication configured for INSERT, UPDATE, DELETE on `public.orders`
