**Priority:** P1
**Status:** Draft

---

## Story

**As a** developer,
**I want** all Firebase data hooks rewritten to use Supabase client with Realtime subscriptions,
**So that** the application reads from and writes to the new PostgreSQL database.

---

## Context

After Story 1.4 decomposes the orders page, the Firebase hooks are isolated in 4 hook files + Server Actions. This story rewrites them to Supabase:
- `useCollection()` → Supabase Realtime channels
- `useDoc()` → Supabase `.select().single()` + Realtime
- Server Actions: Firebase Admin SDK → Supabase service client

The decomposition from Story 1.4 makes this rewrite manageable — each hook is ~80-120 lines instead of embedded in 1,704 lines.

**Reference:** `docs/architecture/technical-debt-assessment.md` §3.3 Phase C

---

## Acceptance Criteria

- [ ] **AC1:** `useOrders` hook rewritten for Supabase
  - Fetches orders from Supabase with `.select()` including joins
  - Real-time subscription via Supabase Realtime channel
  - Handles INSERT, UPDATE, DELETE events
  - Country-scoped queries work correctly
  - Test: Add order in Supabase → UI updates within 2s

- [ ] **AC2:** `useOrderFilters` hook works with Supabase data
  - Search filters applied via Supabase `.ilike()` or client-side
  - Date range filtering works with PostgreSQL TIMESTAMPTZ
  - Status filtering works

- [ ] **AC3:** Server Actions rewritten for Supabase
  - Order CRUD operations use Supabase service client
  - Auth verification via Supabase session (from Story 1.2)
  - RLS policies enforced on all operations
  - Error handling returns appropriate messages

- [ ] **AC4:** Profit stats hooks use Supabase
  - Read from normalized `profit_stats` table
  - Aggregate queries replace single-document reads
  - Historical period support enabled

- [ ] **AC5:** Settings/config reads use Supabase
  - App settings fetched from `app_settings` table
  - Webhook URL read from settings row

- [ ] **AC6:** Duplicate Firebase initialization removed (TD-H2)
  - Remove `src/lib/firebase/` OR `src/firebase/` (consolidate to one)
  - No Firebase client imports remaining in application code
  - Firebase packages still installed (removed in Story 1.9)

- [ ] **AC7:** Supabase client properly configured
  - Environment variables: `NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY`
  - Service role key for Server Actions only (not exposed client-side)
  - Client singleton pattern (no multiple instances)
