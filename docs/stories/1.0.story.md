# Story 1.0: Pre-Migration Critical Security Fixes

**Epic:** EPIC-1 (Firebase → Supabase Quality Migration)
**Sprint:** 0 (Day 0)

**Priority:** P0 — IMMEDIATE
**Status:** Draft
**Agent:** @dev (Dex)

---

## Story

**As a** project owner,
**I want** all active security vulnerabilities fixed immediately,
**So that** order data is protected from unauthorized access and modification while the migration is planned.

---

## Context

The Brownfield Discovery identified 6 critical + 2 high-priority vulnerabilities that are exploitable today. Any logged-in user (including suppliers/FORNECEDOR) can modify any order, redirect webhooks, and access all data. There are no backups. This story closes all active security gaps in the current Firebase system.

---

## Acceptance Criteria

- [ ] **AC1:** Firestore rules restrict order writes to ADMIN role only (TD-C1)
  - Suppliers can read orders for their assigned countries but cannot write
  - Test: Attempt write as FORNECEDOR user → should be denied

- [ ] **AC2:** Firestore rules restrict settings writes to ADMIN role only (TD-C2)
  - Webhook URL and app settings protected from non-admin modification
  - Test: Attempt settings update as FORNECEDOR → should be denied

- [ ] **AC3:** All Server Actions validate auth before executing (TD-C3)
  - Every server action checks `auth().currentUser` and verifies role
  - Admin-only actions reject non-admin users with 403
  - Test: Call each server action without auth token → should fail

- [ ] **AC4:** `ignoreBuildErrors` removed from next.config.js (TD-C5)
  - TypeScript errors now block builds
  - Any existing TS errors are fixed (not suppressed)
  - Test: `npm run build` succeeds without `ignoreBuildErrors`

- [ ] **AC5:** Webhook field name standardized (TD-C6)
  - Consistent field name (`webhookUrl` or `url`) across settings reads and writes
  - No data loss from field name mismatch
  - Test: Update webhook URL in settings → Cloud Function reads correct value

- [ ] **AC6:** Firestore scheduled backups enabled (TD-C8)
  - Daily backup schedule configured in Firebase console or via CLI
  - Backup destination bucket exists and is accessible
  - Test: Verify backup appears in Cloud Storage after scheduled run

- [ ] **AC7:** Cloud Function redeployed to europe-west1 (TD-H5)
  - Function runs in same region as Firestore data
  - Webhook notification latency reduced
  - Test: Trigger tracking update → webhook fires within 2s

- [ ] **AC8:** Typo "avaiable" → "available" fixed (TD-L9)
  - All instances corrected in production code
